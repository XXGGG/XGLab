# 03-数组与元组 🍱

## 👋 嗨！
如果你有很多个苹果，你会怎么放？肯定是一个箱子全装苹果，而不是苹果、鞋子、猫咪混着装，对吧？
在 TypeScript 里，**数组 (Array)** 就是这个箱子。

而**元组 (Tuple)**，则像是一个定制的礼盒，第一个格子放苹果，第二个格子放香蕉，位置和数量都定死了。

---

## 🚃 1. 数组 (Array)

定义数组最简单的方法是在类型后面加个 `[]`。

```typescript
// 1. 字符串数组：只能放字符串
let hobbies: string[] = ["Coding", "Gaming", "Sleeping"];
// hobbies.push(100); // ❌ 报错！不能放数字进去

// 2. 数字数组：只能放数字
let scores: number[] = [98, 95, 100];

// 3. 布尔数组
let answers: boolean[] = [true, false, true];
```

### 另一种写法 (泛型写法)
你可能还会看到这种写法，效果是一模一样的：
```typescript
let names: Array<string> = ["Alice", "Bob"];
```
*小白建议：优先用 `string[]` 这种写法，少打几个字，看着也清爽。*

## 🍱 2. 元组 (Tuple)

元组是 TS 特有的概念（JS 里没有）。
它用于表示一个**已知元素数量和类型**的数组。

场景：比如我们要存一个人的简单信息 `[姓名, 年龄]`。

```typescript
// 定义一个元组：第一个必须是 string，第二个必须是 number
let person: [string, number] = ["Trae", 18];

// person[0] = 123; // ❌ 报错！第一个位置必须是 string
// person[1] = "18"; // ❌ 报错！第二个位置必须是 number

// 越界也不行
// person[2] = "hack"; // ❌ 报错！这个元组只有两个元素
```

**什么时候用元组？**
- CSV 文件的一行数据。
- React Hooks 的返回值（比如 `useState` 返回的就是 `[state, setState]`，这其实就是个元组！）。

---

## 🖼️ Vue 和 React 场景

在框架中，我们经常需要处理列表数据。

### Vue (ref 与数组)
```typescript
import { ref } from 'vue';

// 定义一个响应式的字符串数组
const todoList = ref<string[]>(["吃饭", "睡觉", "打豆豆"]);

// 定义一个对象数组（稍微超纲，下节课细讲对象，先看个大概）
const users = ref<{ id: number; name: string }[]>([
  { id: 1, name: "Alice" }
]);
```

### React (useState 与数组)
```tsx
import { useState } from 'react';

const TodoApp = () => {
  // 泛型 <string[]> 告诉 React，这个 state 是个字符串数组
  const [todos, setTodos] = useState<string[]>(["Learn TS"]);
  
  return (
    <ul>
      {todos.map(todo => <li key={todo}>{todo}</li>)}
    </ul>
  );
};
```
