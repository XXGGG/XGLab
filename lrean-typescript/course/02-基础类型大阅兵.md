# 02-基础类型大阅兵 💂‍♂️

## 👋 欢迎回来！
上一节课如果你成功运行了代码，那你已经击败了 50% 的入门者！今天我们要学习 TypeScript 中最基础、也是最重要的概念——**基础类型**。

想象一下，你有很多个收纳盒。
- 有的盒子只能放 🍎 (String)
- 有的盒子只能放 🔢 (Number)
- 有的盒子只能放 ✅/❌ (Boolean)

TypeScript 就是那个严厉的仓库管理员，它会确保你不会把苹果扔进数字盒子里。

---

## 🧱 1. 三大金刚：string, number, boolean

这三个是 JS 里最常用的，TS 给它们加上了严格的定义。

```typescript
// 1. 字符串 (String) - 文字、单词、句子
let username: string = "XGCode";
// username = 123; // ❌ 报错！不能把数字给字符串

// 2. 数字 (Number) - 整数、小数、负数都是 number
let age: number = 18;
let height: number = 175.5;
// age = "18"; // ❌ 报错！"18" 是字符串

// 3. 布尔值 (Boolean) - 只有 true 或 false
let isHappy: boolean = true;
let isLogin: boolean = false;
// isLogin = 1; // ❌ 报错！虽然 JS 里 1 可以当真，但在 TS 里不行！
```

### 💡 聪明的小贴士：类型推断 (Type Inference)
其实 TS 很聪明。如果你直接赋值，它能猜出类型。

```typescript
let score = 100; // TS 自动推断 score 是 number
// score = "A"; // ❌ 依然会报错！
```
*小白建议：刚开始学习时，建议把 `: type` 写出来，熟悉了以后可以偷懒。*

## 😈 2. 捣蛋鬼：any (任意类型)

有时候你可能会看到这种代码：

```typescript
let anything: any = "hello";
anything = 123;   // ✅ 不报错
anything = true;  // ✅ 不报错
anything.fly();   // ✅ 也不报错（但运行会炸！）
```

`any` 就是告诉 TS：“闭嘴，别管我，我爱咋咋地。”
**千万别滥用 `any`！** 否则你写的就不是 TypeScript，而是 `AnyScript` 了。这失去了用 TS 的意义。

**什么时候用？**
- 只有在你真的不知道（或者不在乎）这个变量是什么类型时，比如处理一些非常动态的第三方数据。

## 👻 3. 还有谁？void, null, undefined

- **void**: 通常用于函数**没有返回值**。
- **null / undefined**: 咱们老熟人了，表示“啥也没有”。

```typescript
function sayHi(): void {
  console.log("Hi!");
  // 这里没有 return，所以是 void
}
```

## 🖼️ Vue 和 React 场景

### Vue (Props 定义)
在 Vue3 中，我们可以给 Props 定义基础类型：

```typescript
// 假设这是子组件
defineProps<{
  title: string;   // 必传，字符串
  count: number;   // 必传，数字
  isShow: boolean; // 必传，布尔值
}>();
```

### React (Props 定义)
React 组件的 Props 也是一样的道理：

```tsx
type Props = {
  name: string;
  age: number;
  isAdmin: boolean;
};

const UserCard = (props: Props) => {
  return <div>{props.name} - {props.age}</div>;
};
```
*是不是发现 Vue 和 React 在 TS 面前，长得越来越像了？*
