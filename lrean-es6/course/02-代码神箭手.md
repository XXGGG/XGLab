# 02-代码神箭手.md

🏹 欢迎来到第二课！

上一节课我们学会了怎么声明变量。今天，我们要升级我们的武器库，学习一种更帅气的函数写法——**箭头函数 (Arrow Function)**。

它长这样：`() => {}`。是不是很像一支穿云箭？

## 🐣 基础用法

### 1. 瘦身前 vs 瘦身后

以前我们写函数：

```javascript
// 传统写法
function add(a, b) {
  return a + b;
}
```

现在我们写函数：

```javascript
// 箭头函数写法
const add = (a, b) => {
  return a + b;
};
```

### 2. 懒人专属福利

如果函数体**只有一行代码**，而且是**返回一个值**，我们可以把 `{}` 和 `return` 都扔掉！

```javascript
// 极简写法
const add = (a, b) => a + b; // 自带 return 功能
```

如果只有**一个参数**，连小括号 `()` 都可以省！

```javascript
const double = x => x * 2;
```

### 3. `this` 的大坑（简单版）

在以前，`this` 指向谁是个玄学。箭头函数最大的好处是：**它没有自己的 `this`**。它会直接沿用外面的 `this`。

简单来说：**写回调函数的时候，用箭头函数通常更安全，不容易出错。**

## 🎭 场景实战

### Vue (Nuxt) 中的用法

在 Vue 中，我们经常在数组方法里用箭头函数。

```javascript
<script setup>
import { computed, ref } from 'vue';

const list = ref([1, 2, 3, 4, 5]);

// 使用箭头函数计算偶数，简洁明了！
const evenNumbers = computed(() => list.value.filter(num => num % 2 === 0));
</script>
```

### React (Next) 中的用法

React 组件其实就是函数，箭头函数在这里无处不在！

```javascript
// 定义组件
const Welcome = (props) => {
  return <h1>Hello, {props.name}</h1>;
};

// 极简写法组件
const SimpleWelcome = ({ name }) => <h1>Hello, {name}</h1>;
```

还有点击事件：

```javascript
<button onClick={() => alert('被点了一下！')}>
  点我
</button>
```

## 🧠 总结

- 看到 `function` 就想能不能换成 `=>`。
- 如果只有一行返回语句，大胆去掉 `{}` 和 `return`。
- 写组件、写回调、写数组操作（map/filter）时，箭头函数是首选。
