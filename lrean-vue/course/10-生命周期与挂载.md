# 10-生命的旅程 🌱

## 👶 生老病死

组件和人一样，也有生命周期。
- 出生 (Creation)
- 挂载 (Mounting) - 也就是显示在屏幕上
- 更新 (Updating) - 数据变了
- 死亡 (Unmounting) - 被移除销毁

在 Vue 3 Composition API 中，我们主要关注一个最重要的时刻：**挂载后 (onMounted)**。

## 1. `onMounted` (我出生啦！)

这是最常用的钩子。
它表示：**组件的 HTML 已经被渲染到浏览器页面上了**。

通常我们在这里做两件事：
1. **找 DOM 元素**（虽然 Vue 不推荐直接操作 DOM，但有时候不得不做，比如用 echarts 画图）。
2. **发送网络请求**（找后端要数据）。

### 🌰 举个栗子

```html
<script setup>
  import { ref, onMounted } from 'vue';

  const list = ref([]);

  // 模拟一个网络请求
  const fetchData = async () => {
    // 假设我们要等 1 秒钟
    setTimeout(() => {
      list.value = ['数据1', '数据2', '数据3'];
      console.log('数据加载完毕！');
    }, 1000);
  };

  // 当组件挂载完成后，自动执行
  onMounted(() => {
    console.log('组件挂载好了！准备去拿数据...');
    fetchData();
  });
</script>

<template>
  <ul>
    <li v-for="item in list" :key="item">{{ item }}</li>
  </ul>
</template>
```

## 2. 其他钩子 (了解即可)

- `onUpdated`: 数据变了，页面也更新完了。
- `onUnmounted`: 组件要死了（比如你切换路由去了别的页面）。这时候通常用来清理定时器，防止内存泄漏。
